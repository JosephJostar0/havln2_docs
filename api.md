# HA-VLN Simulator API Reference

This document details the core API interfaces exposed by the HA-VLN dynamic environment. These interfaces allow developers to bypass the standard black-box reinforcement learning loop and interact directly with the underlying physics engine, human rendering pipeline, and advanced evaluation modules.

Applicable scenarios include, but are not limited to: customizing dense reward functions based on human distance, developing active-following or interactive Agents, and implementing non-standard scene intervention algorithms.

The document is divided into three core modules: **Human State Queries**, **Dynamic Scene Updates**, and **Collision Checks**.

---

## 1. Human State Queries

These APIs are used to obtain real-time spatial topology and visual perception information of surrounding dynamic human models during navigation. Most observation data is exposed through the `info` callback dictionary returned by `env.step()`.

### 1.1 `distance_to_human` (Distance and Relative Angle Query)
* **Description**: Calculates the exact Euclidean distance and relative azimuth angle between the Agent's current position and all rendered active human models in the scene.
* **Configuration Dependency**: `DISTANCE_TO_HUMAN` must be added to the `TASK.MEASUREMENTS` list in `HAVLNCE_task.yaml`.
* **Data Structure**: Returns a list of dictionaries, where each dictionary contains `distance` (float, in meters) and `angle` (float, in radians).
* **Usage Example**:
```python
observations, reward, done, info = env.step(action)

if 'distance_to_human' in info:
    human_states = info['distance_to_human']
    
    # Example return value:
    # [
    #    {"distance": 1.25, "angle": 0.45}, 
    #    {"distance": 5.10, "angle": -1.20}
    # ]
    
    # Application case: Calculate the distance to the nearest human for safety penalties
    min_distance = min([h["distance"] for h in human_states]) if human_states else float('inf')
    if min_distance < 0.5:
        reward -= 5.0  # Trigger safety threshold penalty
```

### 1.2 `_human_posisions` (Absolute Spatial Coordinate Query)
* **Description**: Reads data directly from the underlying physics engine to obtain the absolute 3D coordinates and rotation states of all human models in the Global Map for the current frame. This data completely bypasses the limitations of the Agent's sensor Field of View (FoV).
* **Configuration Dependency**: The `HAVLNCE` utility class must be instantiated, and `ADD_HUMAN: True` must be enabled.
* **Data Structure**: Returns a dictionary. The key is the Viewpoint ID (string), and the value is a two-element tuple: `([x, y, z] coordinate array, [Euler angles] array)`.
* **Usage Example**:
```python
# Assuming env.havlnce_tool is an instance of the HAVLNCE class
global_positions = env.havlnce_tool._sim._human_posisions

# Example return value:
# {
#     "viewpoint_1": (np.array([10.5, 0.1, -4.2]), [0.0, 1.57, 0.0]),
#     "viewpoint_2": (np.array([12.1, 0.1, -3.8]), [0.0, 3.14, 0.0])
# }
```

### 1.3 `human_counting` (In-View Human Visual Counting)
* **Description**: Calls the built-in GroundingDINO object detector to infer the Agent's current RGB observation frame, returning the number of detected humans and outputting a rendered frame with Bounding Boxes.
* **Configuration Dependency**: 
  1. `HUMAN_COUNTING: True` in `HAVLNCE_task.yaml`.
  2. The absolute path to the pre-trained weight file must be correctly configured in `detector.py`.
* **Invocation Method**: Called by directly instantiating the `Detector` and processing the observation dictionary, rather than retrieving it through the `info` dictionary.
* **Usage Example**:
```python
from HASimulator.detector import Detector

detector = Detector().to(device)
stats_info = {}

# current_episodes is a list of Episode objects for the current parallel environments
detected_imgs = detector(observations, 'human', current_episodes, stats_info)

# The stats_info dictionary will be updated in-place with the count information for each environment
# detected_imgs contains RGB numpy arrays with drawn bounding boxes, which can replace original observations
```

## 2. Dynamic Scene Updates

The timeline of the HA-VLN environment is controlled by a child thread clock. The following APIs allow developers to manage clock signals or forcibly manipulate the timeline and underlying physics mesh during debugging or special tasks.

### 2.1 `_handle_signals()` (Thread-Safe Time Stepping)
* **Description**: The core API for environment synchronization. It actively consumes the time pulse signals generated by the child thread and triggers the complete physical update lifecycle (remove old human models -> insert new human models -> load/recompute NavMesh).
* **Usage Standard**: In a standard RL or planning loop, this method **must** be called before `sim.step()` is executed to ensure the Agent's actions are based on the latest physical world state.
* **Usage Example**:
```python
# Called inside the Wrapper's step() method
self.havlnce_tool._handle_signals()
```

### 2.2 `refresh_human_model(frame_id)` (Forced Frame Jump)
* **Description**: Interrupts the standard time progression, forcing the physics engine to remove the currently rendered human poses and load the 3D model configuration corresponding to a specific frame ID.
* **Parameters**: `frame_id` (int), valid range is typically `0` to `119` (corresponding to the 120 frames of the SMPL motion sequence).
* **Usage Example**:
```python
# Forcibly jump to the middle state of the motion sequence (frame 60)
target_frame = 60
env.havlnce_tool.refresh_human_model(target_frame)
```

### 2.3 `recompute_navmesh(...)` (Low-Level NavMesh Recomputation)
* **Description**: Manually triggers the Habitat engine's NavMesh recomputation logic. When dynamic objects (humans) in the scene move and no cached `.navmesh` file is available, this API must be called to reconstruct the physical collision volume.
* **Performance Warning**: This operation is extremely CPU-intensive and should rely on cached mesh files during real-time training. It is only recommended to call this manually when generating new trajectories or testing new scenes.
* **Usage Example**:
```python
import habitat_sim

navmesh_settings = habitat_sim.nav.NavMeshSettings()
navmesh_settings.set_defaults()
navmesh_settings.agent_radius = 0.1  # Set Agent collision radius
navmesh_settings.agent_height = 1.5  # Set Agent collision height

# Execute recomputation
success = env.havlnce_tool._sim.recompute_navmesh(
    env.havlnce_tool._sim.pathfinder, 
    navmesh_settings, 
    include_static_objects=True
)

if success:
    env.havlnce_tool._sim.pathfinder.save_nav_mesh("path/to/save.navmesh")
```

## 3. Collision Checks

This module goes beyond Habitat's default binary collision recording (collided/did not collide) by providing object-level collision traceability and advanced evaluation tools calibrated for Matterport3D scene artifacts.

### 3.1 `collisions_detail` (Fine-Grained Collision Recording)
* **Description**: Returns the types and descriptions of all physical entities the Agent collided with during the current step, used to distinguish between static environment collisions and dynamic human collisions.
* **Configuration Dependency**: `COLLISIONS_DETAIL` must be added to the `TASK.MEASUREMENTS` list in `HAVLNCE_task.yaml`.
* **Usage Example**:
```python
observations, reward, done, info = env.step(action)

if 'collisions_detail' in info:
    details = info['collisions_detail']
    
    # Check collision types based on the specific return structure
    # Assuming 'details' returns a list or dictionary containing object names
    if any("human" in str(obj).lower() for obj in details):
        print("Warning: Human collision detected!")
        done = True  # Trigger severe violation termination
```

### 3.2 `Calculate_Metric` (Offline Trajectory Collision Rate / TCR Calculation)
* **Description**: Based on the official pre-extracted Oracle path data, it excludes unavoidable basic environmental collisions in complex scenes to calculate the true "net new collision rate" (i.e., the Trajectory Collision Rate, TCR, against dynamic humans) and the strict Success Rate (SR).
* **Usage Standard**: Typically called offline for global statistical calculation of an Episode after it has finished running (Done).
* **Usage Example**:
```python
from HASimulator.metric import Calculate_Metric

# 1. Instantiate (specify the current evaluation split, e.g., 'train', 'val_seen', 'val_unseen')
metric_calc = Calculate_Metric(split="val_unseen")

# 2. After the Episode ends, execute metric injection
# 'info' is the accumulated metric dictionary for the Episode, 'episode_id' is the current task ID
metric_calc(info, episode_id)

# 3. Extract the calculated core metrics
tcr = info['TCR']       # Trajectory Collision Rate (collisions excluding the baseline)
cr = info['CR']         # Collision Rate (Boolean: 0 or 1)
strict_sr = info['SR']  # Strict Success Rate under the premise of zero TCR
```
